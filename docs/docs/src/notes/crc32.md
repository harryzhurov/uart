# Памятка CRC-32

## Ethernet

### Образующий полином

Formula: \(x32+x26+x23+x22+x16+x12+x11+x10+x8+x7+x5+x4+x2+x+1\)
<br>
hexadecimal: `0x04C11DB7`

[Краткое описание и примеры](https://fastercapital.com/topics/exploring-the-algorithm.html).


### Получение данных из Wireshark

Получить данные из Wireshark можно двумя способами:

1. скопировав hex-строку непосредственно из программы;
2. экспортировав данные пакета в файл.

#### Копирование строки

Для этого нужно выделить весь пакет&nbsp;– для этого нужно встать на верхнуюю строку в свойствах пакета (начинается с `Frame 1:...`), при этом в окне дампа будет выделен весь буфер. Далее вызов контекстного меню с помощью нажатия на правую кнопку мыши (ПКМ), в нём `Copy->...as a HEX Stream`. 

hex-строка в буфере обмена, можно вставить, куда нужно&nbsp;– например, во вводное окно онлайн-калькулятора.

#### Экспорт в файл

Для этого нужно также выделить требуемый пакет, далее в том же контекстном меню выбрать пункт `Export Packet Bytes...`, в появившемся диалоговом окне указать имя файла. Данные (октеты) пакета сохраняются в двоичном "сыром" (raw) виде.

## Калькуляторы

### Online Calculator, вариант 1

[Годный онлайн калькулятор](https://www.lddgo.net/en/encrypt/crc), позволяющий вводить как hex-строку (можно скопировать из Wireshark), так и указать двоичный файл.

Для CRC-32 Ethernet следует выбрать алгоритм CRC-32, контролировать можно по коду образующего полинома.
### Online Calculator, вариант 2

[Ещё один годный калькулятор](https://crccalc.com/?crc=&method=crc32&datatype=1&outtype=0), позволяет только вставлять hex-строку.

Для получения результата CRC-32 Ethernet, следует смотреть вариант `CRC-32/ISO-HDLC`, контролировать можно по коду образующего полинома.

### Программа на Python

Код программы оформлен в виде функции:

```python
import struct
import binascii as bs

def fcs(s):
    b = bs.unhexlify(s)
    crc   = bs.crc32( b ) & 0xffffffff
    crc_s = struct.pack('I', crc)
    return hex(crc), bs.hexlify( crc_s)
```

Функция принимает строку в формате hex-строки, например:

```python
'c83a35d8c5df000000000001080045000030000140004011a568c0a80a02c0a80a01c350c350001c00000001000000020003000400050006000700080009'
```

Результат выдаётся в двух форматах: как 32-разрядное число в шестнадцатеричном представлении и как поток байт в little-endian представлении. 

Пример:

```python
In [50]: fcs('c83a35d8c5df000000000001080045000030000140004011a568c0a80a02c0a80a01c350c350001c00000001000000020003000400050006000700080009')
Out[50]: ('0xa110d73f', b'3fd710a1')
```

## Проверка результата

При вычислении результата CRC-32 потока октетов вместе с самой контрольной суммой (типовая ситуация при приёме Ethernet фрейма) результат равен некоему "магическому числу". В рассматриваемом случае это число равно:

```
0xdebb20e3
```

Именно с этим числом и нужно сравнивать результат вычислений: если ошибок нет, то результат будет равен этому значению.
