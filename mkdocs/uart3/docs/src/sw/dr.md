## Драйвер

Драйвер разбит на две части **tx_driver** и **rx_driver**, которые работают параллельно и независимо. 

Каждый из поддрайверов разбит на задачи (**task**), которые преобразуют рандомизированные данные в синхронизированную передачу сигналов для RTL модуля UART или считывают данные с него и записывают в буфер. 

### tx_driver

Поддрайвер **tx_driver** разбит на два независимых блока: передатчик и приемник. 

Передатчик формирует задержку `data_delay`, если `send_del = 1`. Затем передает данные от рандомизатора **txRandomizer** на входе `tx_data` модуля UART, если `tx_empty = 1`, после чего формирует *лог.1* на входе `tx_wren` длительность 1 такт `clk`. Когда UART возвращает флаг `tx_complete = 1`, процесс повторяется.

Приемник ожидает появления стар-бита на линии `txc` и начинает записывать данные в сдвиговый регистр. По окончании транзакции складывает полученные данные в буфер.

### rx_driver

Поддрайвер **rx_driver** разбит на три независимых блока: передатчик, приемник, формирователь флага `rx_rden`.

Передатчик выжидает время, равное `send_delay`, если `send_del = 1`. оиждает синхроимпульса `baud_pulse` и осуществляет отправку данных на линию `rxc`. Если стоп-бит = *лог.0*, производится проверка флага `frame_error`, после чего отправляется сигнал обнуления ошибок `rst_err`. 

Приемник ожидает положительного фронта флага `rx_complete` и вычитывает данные с шины `rx_data` в буфер. Если подразумевается наличие флага `overrun` (`rx_complete` не сброшен), то считывание данных происходит при появлении положительного фронта флага `overrun`, чтобы не терять данные.

Формирователь флага `rx_rden`, ожидает положительного фронта `rx_complete`, формирует задержку `rden_delay`, если `wrong_rden = 1`, после чего делает проверку на `overrun`, сравнивая величину задержки с длительностью отправки одного пакета. Если обнаруживается флаг `overrun`, отпарвляется сигнал сброса ошибок `rst_err`. Отправляется сигнал `rx_rden` длительностью один такт клока.
